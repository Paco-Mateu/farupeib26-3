# Data Workspace

Use this folder to stage the data that the prototype will need during the contest.

- `raw/`: downloaded public datasets, exports, CSVs, PDFs, JSON, and source snapshots.
- `processed/`: cleaned, normalized, or chunked data that is ready for MongoDB import or embedding.
- `synthetic/`: generated demo records, fixtures, and fallback content for fast prototypes.

Keep small sample files in git when they help onboarding. Keep large event-only files out of git when they are disposable.
